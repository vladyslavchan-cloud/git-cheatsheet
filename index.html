Git & GitHub Cheatsheet

Your Magical Notebook for Version Control

üõ†Ô∏è Initial Setup (One-Time Only)

Configure this once per computer to identify who you are. This is the "WHO" in version control history.
git --version Check if Git is installed
sudo apt-get install git Install Git (Linux System)
git config --global user.name "Name" Set your username
git config --global user.email "mail@ex.com" Set email (must match GitHub!)
git config --global init.defaultBranch main Set default branch to "main"
ssh-keygen -t ed25519 -C "mail@ex.com" Generate SSH Key
eval "$(ssh-agent -s)" Start SSH Agent
ssh-add ~/.ssh/id_ed25519 Add SSH key to Agent
cat ~/.ssh/id_ed25519.pub Show SSH Key (For copying)
Copy the SSH key (no whitespace after ".com") and paste into GitHub Add SSH key to GitHub
ssh -T git@github.com Test SSH connection to GitHub

üîÑ The Daily Workflow

Think of the 3 zones: Working Directory (Store) → Staging Area (Cart) → Repository (Fridge).
git init Start a new repo (create .git folder)
git status Check what has changed
git add <file> Stage specific file (Fill the cart)
git add . Stage ALL changes (Fill the cart)
git commit -m "Message" Save snapshot (Checkout)
git remote add origin <url> Connect local repo to GitHub (Remote repo URL - only for initial connection).
git push -u origin main Upload to GitHub (First time)
git push Upload changes (After initial connection and push -u)

‚è™ Safe Undoing

Fix mistakes without destroying history. Avoid dangerous resets for now!
git restore <file> Discard local changes (Unsaved)
git restore --staged <file> Unstage file (Keep changes)
git commit --amend Fix last commit message
git reset --soft HEAD~1 Undo commit, keep changes staged

üåø Branching

Work on features in isolation without endangering the "main" trunk.
git branch List all local branches
git switch -c <name> Create & switch to new branch
git switch <name> Switch to existing branch
git merge <branch> Merge branch into current one
